Settings 配置页 — UI 设计方案

基于当前 Models / Skills / Rules / Subagents / Tools 功能需求与 coss 规范（阅读栏 52rem、gap/p token、rounded-lg 卡片、语义色）。点击上方标签切换方案，在 mock 壳层内点击导航体验交互。

设计思路

保留左侧 Explorer 导航 + 右侧单列内容，在现有 ExpandableCard 基础上做一致性打磨：统一阅读栏宽度、区分「可编辑 / 只读」、Models 增加连接状态、折叠卡片默认只显示一行摘要。

适用场景

  • 改动最小，与现有 settings-editor.tsx 一脉相承
  • 条目少（<10）时足够清晰
  • 适合先 ship 再迭代

Settings 首页为 Hub：每张卡片展示类别、数量、一句摘要，点击进入二级页。Explorer 侧栏可简化为单一「Settings」入口，减少五栏常驻占用。

Quick tip
在 Chat 输入区可切换当前模型；Skills 与 Rules 修改后需重启 Agent。
当前实现的主要问题： (1) Models 用扁平卡片 + 内联表单，其余用 ExpandableCard accordion，交互模型不统一； (2) 只读插件项与可编辑 Models 视觉区分不足； (3) Rules/Subagents 的 preview 藏在折叠里，阅读成本高； (4) 无搜索/过滤，条目增多后可扫性下降； (5) Skills 与 Subagents 的从属关系未表达。
维度 A 现状优化 B 主从 Inspector C 密集目录表 D 能力树 E Hub 总览
实现成本低中中中高中
阅读长 preview一般好需弹层好需二级
条目扩展性<10好最好好一般
表达插件层级无无Source 列最好无
Models CRUD自然Inspector 表单表格行分离区块二级页
新用户友好一般一般偏专业好最好
符合 coss 规范是是是是是

推荐组合： 短期用 A 统一阅读栏与可编辑/只读标签； 中期对 Skills/Rules/Subagents 采用 B 主从布局（Models 可保留 A 的卡片或迁入 B 的 Inspector）； 若插件数量持续增长，C 的 Segmented + Table 可作为「Advanced」视图； D 适合调试向用户；E 可作为进入 Settings 的 landing，再 drill-down 到 B/A。
