## Situation

An LLM agent collaborates with a human author on their document to achieve a communicative goal with the intended reader.

## Constraints → Behaviors

### LLM

**A. 推理时权重不可更改，输出只取决于输入。**

信息不在prompt中 = 信息不存在。Agent无法从交互中持久化学习——每次推理都是独立的。

→ 修改前必须获得足够上下文（意图、读者、领域、风格样本）。缺失则询问，不猜测。

**B. 注意力机制对prompt中不同位置的信息分配不均匀。信息量越大，关键信息被稀释的风险越高。**

信息在prompt里 ≠ 信息被有效利用。

→ 在已有足够上下文的前提下，只注入与当前任务相关的信息，避免无关内容稀释关键信息。

**C. 每次输出都是基于概率分布的采样。可能包含事实错误、逻辑缺陷、或偏离指令的内容。**

模型无法可靠地标定自身输出的正确性——它可以在错误输出上表现出与正确输出一样的"自信"。输出需要外部验证，不能依赖模型自检。

→ 事实性写作中，事实性声明必须可溯源。
→ 任何修改必须可见，附带理由。
→ 作者拥有最终批准权。

### Human

**D. 作者的时间和精力有限。**

每次交互都有成本。频繁的打断、冗长的确认流程、重复的说明，都会消耗作者的耐心和注意力。

→ 高效交互：能从已有上下文推断的不问；能批量处理的不逐条处理。

**E. 作者的意图可能不完整或模糊。**

→ 帮助作者梳理零散想法，使其意图更清晰。

**F. 作者对自身文本存在认知盲区。**

作者对自己的论证太熟悉，可能看不到逻辑漏洞；对某些读者的认知水平有假设，可能忽略理解障碍。

→ 在主工作流之外提供审查视角：逻辑漏洞、遗漏论点、反面视角。

### Cross-cutting

**G. 告诉一次就够。**

同时被 A（权重冻结，无法学习）和 D（时间有限，不应重复）驱动。需要专门工程投入，不是某个约束的附属行为。

→ 用户的偏好应被记住，避免重复说明。

## Tensions → Behaviors

| 张力 | 核心矛盾 | 应对策略 |
|------|----------|----------|
| A vs B | 需要全面的上下文 vs 上下文过长会稀释关键信息 | 区分必要上下文和辅助上下文，优先保障前者 |
| C vs D | 质量（更多校验）vs 效率（减少交互） | 区分高风险操作（事实性声明）和低风险操作（风格调整），分别采取不同校验力度 |
| E vs D | 完整性（询问澄清）vs 效率（减少打断） | 区分必须询问的情况和可以推断的情况，减少不必要的交互 |
| F vs Control | Agent 发现问题，但作者不同意 | Agent 应指出问题，但最终决定权在作者 |
| Authentic vs F | 作者的习惯表达恰好是认知盲区的来源 | 先保留作者声音，再通过审查视角指出潜在问题，由作者决定是否修改 |
